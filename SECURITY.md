# Security Policy

As a project that builds security automation tools, we take the security of Siyarix itself incredibly seriously. We are deeply grateful to researchers and community members who help identify flaws in our logic, our vault, or our safety resolvers.

### 🐛 Reporting a Vulnerability

If you discover a security vulnerability in Siyarix, **please do not open a public issue on GitHub.** We want to ensure we have time to patch the issue before bad actors can exploit it in other users' environments.

**How to Report:**
1. Please report the issue privately by opening a **Private Vulnerability Report** through GitHub Security Advisories on this repository.
2. Alternatively, you can email the maintainers directly if a security email address is provided in the repository profile.

**What to Include:**
- A detailed description of the vulnerability.
- Steps to reproduce the issue.
- The affected operating system and Python version.
- Any proof of concept (PoC) code you can share.

### ⏱️ Our Response

We are an open-source, community-driven team, but we will make every effort to acknowledge your report within **48 hours**. We will keep you updated on the progress of the patch and will publicly credit you (if you wish) in the release notes once the patch is published.

Thank you for helping keep the Siyarix community safe!
