| Channel | Purpose | Response Time |
|---------|---------|---------------|
| **GitHub Issues** | Bug reports, feature requests | 2–5 business days |
| **GitHub Discussions** | Questions, ideas, community help | Community-driven |
| **Pull Requests** | Code contributions | Reviewed within 1 week |

---

## Before Asking for Help

1. **Check the documentation** — Read through the relevant docs in the repository.
2. **Search existing issues** — Your question may already have been answered.
3. **Run diagnostics** — Try `siyarix health` and `siyarix shell doctor` for common issues.
4. **Review logs** — Check `~/.siyarix/logs/` for error details.

---

## Filing a Support Request

When opening a support issue, please include:

- Siyarix version (`siyarix --version` or `pip show siyarix`)
- Operating system and Python version
- What you were trying to do
- What happened (include error output, sanitized)
- What you expected to happen
- Steps to reproduce (if applicable)
